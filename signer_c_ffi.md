# Plan: Add signer-in-context support

## Context

The `gpeacock/cawg_embeddable` branch on c2pa-rs introduces builder methods
that obtain the signer from the Builder's Context rather than accepting it as
a parameter. The Rust SDK already has `Context::with_signer()` /
`Context::set_signer()` and `Builder::save_to_stream()` (which calls
`self.context.signer()`), but the C FFI layer exposes **none of these**.

C/C++ users who create signers via callback or credentials have no way to put
their signer into a Context, and there is no signerless `builder.sign()`
overload in the C++ wrapper. Headers are autogenerated by cbindgen in c2pa_c_ffi.

**Goal:** Non-breaking, additive APIs in both repos so that:

1. FFI: transfer a `C2paSigner` into a `C2paContextBuilder` before building
2. FFI: sign via context signer (no explicit signer parameter)
3. C++: `ContextBuilder::with_signer(Signer&&)`, convenience `Context` ctor
4. C++: `Builder::sign()` overloads that use the context signer
5. E2e tests that sign with context signer and verify the output
6. Settings-based signer tests that validate interchangeability
7. Programmatic signer takes priority over settings-based signer
8. CallbackSigner works identically with context APIs (no special handling)

---

## Design notes

### Error behavior

`c2pa_builder_sign_context` follows the exact same error handling pattern as
`c2pa_builder_sign` (`c_api.rs:1529`):

- Same return type (`i64`): positive byte count on success, `-1` on error
- Same macro usage: `deref_mut_or_return_int!`, `cstr_or_return_int!`,
  `ptr_or_return_int!`, `ok_or_return_int!`
- Same `manifest_bytes_ptr` handling: `ptr_or_return_int!` validates non-null,
  then a secondary `is_null()` guard before writing the pointer
- Same error propagation: errors flow through `CimplError::from(e).set_last()`
  and are retrievable via `c2pa_error()`
- When the context has no signer, `save_to_stream` calls `self.context.signer()`
  which returns `Error::MissingSignerSettings` -- this propagates through
  `ok_or_return_int!` just like any signing error in `c2pa_builder_sign`

The only difference is the absence of `signer_ptr` -- that is the purpose.

### Signer priority

The Rust `Context` uses a `SignerState` enum (`context.rs:44`):

- `Custom(BoxedSigner)` -- set by `set_signer()` / `with_signer()`
- `FromSettings(OnceLock<Result<BoxedSigner>>)` -- lazy-initialized from
  `self.settings.signer` on first call to `context.signer()`

`context.signer()` matches `Custom` first (`context.rs:513`). The `set_signer`
method unconditionally replaces the signer field with `Custom` (`context.rs:440`),
while `set_settings` only updates `self.settings` without touching the signer
field (`context.rs:287`).

This means:
- If both settings-based and programmatic signers are configured, the
  programmatic one always wins, regardless of call order.
- `with_settings(...)` then `with_signer(...)` -- programmatic wins
- `with_signer(...)` then `with_settings(...)` -- programmatic still wins
  (because `set_settings` does not reset the signer field)

Tests verify this invariant at both the FFI and C++ layers.

### CallbackSigner integration

The `CallbackSigner` (`sdk/src/callback_signer.rs:33`) is a Rust struct that
implements the `Signer` trait by delegating to a C function pointer callback
with an opaque context. It is created via `c2pa_signer_create()` in
`c_api.rs:1738`.

**No special handling is needed.** At the FFI level, all signers are uniform:

- `c2pa_signer_create()` produces a `*mut C2paSigner` (callback-based)
- `c2pa_signer_from_info()` produces a `*mut C2paSigner` (credential-based)

Both wrap `Box<dyn c2pa::Signer + Send + Sync>` (`c_api.rs:204`).
`c2pa_context_builder_set_signer` accepts any `*mut C2paSigner` pointer and
extracts `.signer` from it -- it does not inspect the concrete type.

In C++, the `Signer` class wraps `C2paSigner*` regardless of how it was
constructed (callback ctor at `c2pa.hpp:734` or credential ctor at
`c2pa.hpp:750`). The `release()` method and `with_signer(Signer&&)` work
identically for both signer types.

The `CallbackSigner` has `unsafe impl Send` and `unsafe impl Sync`
(`callback_signer.rs:53-54`), satisfying the trait bounds for
`Context::set_signer`.

Tests demonstrate that callback-based signers work with the context APIs by
wrapping a credential-based signer inside a callback function, avoiding any
dependency on openssl in unit tests.

### Memory safety and security critique

**Issue 1 -- Pointer leak on FFI failure in C++ `with_signer`:**

In `ContextBuilder::with_signer(Signer&&)`, after `signer.release()` extracts
the raw `C2paSigner*`, if `c2pa_context_builder_set_signer()` were to return
non-zero, the raw pointer would be leaked -- it was released from the source
`Signer` but never freed.

In practice, the underlying `Context::set_signer()` (`context.rs:436-442`)
always returns `Ok(())` and cannot fail. The `ok_or_return_int!` macro in the
FFI wrapper would only trigger on a Rust error, which is currently unreachable.

Nevertheless, as defensive practice, the implementation re-wraps the raw
pointer in a `Signer` RAII guard before throwing, so that even if the Rust API
changes in the future, the pointer is always cleaned up:

```cpp
if (c2pa_context_builder_set_signer(context_builder, raw) != 0) {
    Signer cleanup(raw);   // RAII cleanup if FFI ever fails
    throw C2paException();
}
```

**Issue 2 -- Double-free prevention at the C FFI layer:**

After `c2pa_context_builder_set_signer` consumes the `signer_ptr` via
`Box::from_raw`, the caller must not call `c2pa_signer_free` or `c2pa_free`
on that pointer. The safety documentation explicitly states "MUST NOT be used
or freed" to prevent double-free. This is the same ownership-transfer pattern
used by `c2pa_context_builder_build` which consumes the builder pointer.

**Security review (all safe):**

- Format strings: `cstr_or_return_int!` uses `CStr::from_ptr` which is
  null-terminated and length-bounded. No format string injection is possible.
- `set_signer` does not parse or evaluate the signer -- it only stores it.
  There is no deserialization or code execution at the point of transfer.
- TOCTOU: Not exploitable because the signer pointer is consumed atomically
  via `Box::from_raw` within the same FFI call.
- Callback signers: The `unsafe impl Send/Sync` on `CallbackSigner` is an
  existing assumption in the codebase. Our changes do not weaken or alter
  this guarantee.

```cpp
/// @brief Callback function that delegates signing to a credential-based signer.
/// @details This wraps a credential signer inside a SignerFunc callback, allowing
///          tests to exercise the CallbackSigner path without depending on openssl.
///          The callback reads data from the input buffer, signs it using the
///          credential signer, and returns the signature bytes.
///
///          Usage:
///            c2pa::Signer signer(
///                &credential_callback_signer,
///                C2paSigningAlg::Es256,
///                certs_pem,
///                "http://timestamp.digicert.com");
///
/// @param data Pointer to data to sign.
/// @param len Length of data in bytes.
/// @param signed_bytes Output: pointer set to the allocated signature bytes.
///        Caller (the C library) takes ownership and frees this via c2pa_free.
/// @return Length of the signature in bytes, or -1 on error.
inline intptr_t credential_callback_signer(
    const unsigned char *data, uintptr_t len,
    unsigned char **signed_bytes)
{
    // Build a temporary credential signer to perform the actual signing
    auto fixtures_dir = get_fixture_path("");
    auto certs = read_text_file(fixtures_dir / "es256_certs.pem");
    auto private_key = read_text_file(fixtures_dir / "es256_private.key");
    auto info = C2paSignerInfo {
        "Es256",
        certs.c_str(),
        private_key.c_str(),
        nullptr
    };

    auto *temp_signer = c2pa_signer_from_info(&info);
    if (!temp_signer) {
        return -1;
    }

    // Use c2pa_signer_sign to perform the actual signing
    auto result = c2pa_signer_sign(temp_signer, data, len, signed_bytes);
    c2pa_free(temp_signer);
    return result;
}
```

## Part 1: c2pa-rs FFI changes

**File:** `c2pa_c_ffi/src/c_api.rs`

### 1a. `c2pa_context_builder_set_signer` (new)

Add after `c2pa_context_builder_set_settings` (~line 463):

```rust
/// Transfers ownership of a signer into the context builder.
///
/// After this call the `signer_ptr` is consumed and MUST NOT be used or freed.
/// The signer will be available via `context.signer()` after building.
///
/// # Safety
/// * `builder` must be a valid C2paContextBuilder pointer (not yet built).
/// * `signer_ptr` must be a valid C2paSigner pointer. It is consumed by this call.
#[no_mangle]
pub unsafe extern "C" fn c2pa_context_builder_set_signer(
    builder: *mut C2paContextBuilder,
    signer_ptr: *mut C2paSigner,
) -> c_int {
    let builder = deref_mut_or_return_int!(builder, C2paContextBuilder);
    let c2pa_signer = Box::from_raw(signer_ptr);   // takes ownership
    let result = builder.set_signer(c2pa_signer.signer);
    ok_or_return_int!(result);
    0
}
```

Why it works: `C2paSigner.signer` is `Box<dyn Signer + Send + Sync>`
(`c_api.rs:204`). `Context::set_signer` accepts it because the blanket impls
at `signer.rs:281`, `maybe_send_sync.rs:53,86` satisfy the bounds.
`Box::from_raw` consumes the pointer (same pattern as
`c2pa_context_builder_build` at `c_api.rs:484`).

### 1b. `c2pa_builder_sign_context` (new)

Add after `c2pa_builder_sign` (~line 1556). This wraps Rust's
`Builder::save_to_stream()` which calls `self.context.signer()`:

```rust
/// Signs the builder using the signer from its Context.
///
/// Equivalent to `c2pa_builder_sign` but the signer comes from the
/// builder's context instead of being passed explicitly.
/// Error behavior is identical to `c2pa_builder_sign`: returns -1 on error,
/// sets the error string retrievable via `c2pa_error()`.
/// If the context has no signer (neither programmatic nor from settings),
/// the error will be `MissingSignerSettings`.
///
/// # Parameters
/// * builder_ptr: pointer to a Builder whose context has a signer set.
/// * format: MIME type or file extension.
/// * source: pointer to a readable C2paStream.
/// * dest: pointer to a read+write+seek C2paStream.
/// * manifest_bytes_ptr: out-pointer for the manifest bytes.
///
/// # Safety
/// Reads from NULL-terminated C strings.
/// The returned bytes MUST be released by calling c2pa_free.
#[no_mangle]
pub unsafe extern "C" fn c2pa_builder_sign_context(
    builder_ptr: *mut C2paBuilder,
    format: *const c_char,
    source: *mut C2paStream,
    dest: *mut C2paStream,
    manifest_bytes_ptr: *mut *const c_uchar,
) -> i64 {
    let builder = deref_mut_or_return_int!(builder_ptr, C2paBuilder);
    let format = cstr_or_return_int!(format);
    let source = deref_mut_or_return_int!(source, C2paStream);
    let dest = deref_mut_or_return_int!(dest, C2paStream);
    ptr_or_return_int!(manifest_bytes_ptr);

    let result = builder.save_to_stream(&format, &mut *source, &mut *dest);
    let manifest_bytes = ok_or_return_int!(result);
    let len = manifest_bytes.len() as i64;
    if !manifest_bytes_ptr.is_null() {
        *manifest_bytes_ptr = to_c_bytes(manifest_bytes);
    }
    len
}
```

`C2paStream` implements `Read + Write + Seek + Send` (`c2pa_stream.rs:116-240`),
satisfying `save_to_stream`'s `R: Read + Seek + Send` and
`W: Write + Read + Seek + Send` bounds.

```cpp
/// @details Moves the signer into the context. After this call the

///          takes priority regardless of call order.
/// @param signer Signer to move into the context.
/// @return Reference to this ContextBuilder for method chaining.
/// @throws C2paException if the builder or signer is invalid.

```

**File:** `src/c2pa.cpp`, after `ContextBuilder::with_json_settings_file` (~line 483)

```cpp
Context::ContextBuilder& Context::ContextBuilder::with_signer(Signer&& signer) {
    if (!is_valid()) {
        throw C2paException("ContextBuilder is not valid");
    }
    C2paSigner* raw = signer.release();
    if (!raw) {
        throw C2paException("Signer is not valid");
    }
    if (c2pa_context_builder_set_signer(context_builder, raw) != 0) {
        Signer cleanup(raw);   // RAII cleanup if FFI ever fails
        throw C2paException();
    }
    return *this;
}
```

**Memory safety note:** The `Signer cleanup(raw)` line ensures that even if
`c2pa_context_builder_set_signer` fails (currently unreachable since
`set_signer` always returns `Ok(())`), the raw pointer is properly freed via
the `Signer` destructor. Without this guard, the pointer would leak because
`signer.release()` already nulled the original `Signer` object.

```cpp
std::vector<unsigned char> Builder::sign(
    const std::string& format, std::istream& source, std::iostream& dest) {
    CppIStream c_source(source);
    CppIOStream c_dest(dest);
    const unsigned char* c2pa_manifest_bytes = nullptr;

    auto result = c2pa_builder_sign_context(
        builder, format.c_str(),
        c_source.c_stream, c_dest.c_stream,
        &c2pa_manifest_bytes);
    return detail::to_byte_vector(c2pa_manifest_bytes, result);
}

std::vector<unsigned char> Builder::sign(
    const std::filesystem::path& source_path,
    const std::filesystem::path& dest_path) {
    std::ifstream source(source_path, std::ios::binary);
    if (!source) {
        throw std::system_error(
            errno, std::system_category(),
            "Failed to open source: " + source_path.string());
    }
    std::fstream dest(dest_path,
        std::ios::in | std::ios::out | std::ios::binary | std::ios::trunc);
    if (!dest) {
        throw std::system_error(
            errno, std::system_category(),
            "Failed to open dest: " + dest_path.string());
    }
    return sign(detail::extract_file_extension(source_path), source, dest);
}
```

## Implementation notes

### Deviations from plan

The following adjustments were made during implementation:

1. **Callback signer helper uses `c2pa_ed25519_sign` instead of
   `c2pa_signer_sign`**: The plan proposed wrapping a credential-based signer
   inside a `SignerFunc` callback via `c2pa_signer_sign`. The actual
   implementation uses the simpler `c2pa_ed25519_sign` C API function, which
   takes raw data and a PEM key and returns 64-byte Ed25519 signatures directly.
   This avoids creating a temporary `C2paSigner` in the callback and simplifies
   memory management. The callback helper signature uses the C++ `SignerFunc`
   type (`std::vector<unsigned char>(const std::vector<unsigned char>&)`)
   rather than the raw C callback signature.

2. **FFI test API corrections**: The plan showed `c2pa_settings_new(json, format)`
   and `c2pa_settings_free(settings)`. The actual API is `c2pa_settings_new()`
   (no arguments) followed by `c2pa_settings_update_from_string(settings, json,
   format)`, and settings are freed via `c2pa_free(settings as *mut c_void)`.
   Also, `c2pa_builder_with_definition` returns a `*mut C2paBuilder`
   (consuming the old pointer) rather than a `c_int`.

3. **C++ test naming**: Some test names were shortened for clarity:
   `ProgrammaticSignerOverridesSettingsSignerReversedOrder` became
   `ProgrammaticSignerBeforeSettingsStillWins`, and
   `ConvenienceConstructorWithSettingsSignerOverride` became
   `ConvenienceCtorProgrammaticOverridesSettings`.

4. **docs/context.md signer section**: The plan did not specify documentation
   updates. The implementation rewrote the "Configuring a signer" section
   with all three signer types (settings-based, programmatic, callback),
   signer priority rules, a resolution flowchart, a signing sequence diagram,
   and HTML comments linking each example to its backing test.
